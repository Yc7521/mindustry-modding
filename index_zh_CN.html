1 概览

Mindustry的mod是一些内容物的整合，你可以用内置的种类，改变其中的数据来加载你的物品，也可以重写原版的内容

你可以添加自己的贴图和音效，打包上传并分享自己的mod，mod在所有平台上都是同一格式。如果你确实想用 GitHub, 你需要按照Anuken的格式布置: https://github.com/Anuken/ExampleMod

你也可以添加蓝图到mod里面

1.1 文件结构

你的mod文件结构必须类似于如下结构: 

mod名
├── mod.json
├── content
│   ├── items
│   ├── blocks
│   ├── mechs
│   ├── liquids
│   ├── units
│   └── zones
├── maps
├── bundles
├── sounds
├── scripts
├── schematics
├── sprites-override
└── sprites

(只有 mod.json 是必须)

每个平台都有不一样的数据存储目录，存储目录的mods文件夹就是加载mod的地方:

  • Linux: ~/.local/share/Mindustry/mods/
  • Steam: steam/steamapps/common/Mindustry/mods/
  • Windows: %appdata%/Mindustry/mods/
  • Apple: ~/Library/Application Support/Mindustry/mods/

Note; 你的文件名必须全小写而且不能有空格:

  • 正确: my-custom-block.json
  • 不正确: My Custom Block.json

1.3 mod.json

在mod的根目录，你必须要有 mod.json 这个文件来定义你的mod的基础数据。 

name: Mod Name
author: Yourself
description: This is a useless description.
version: "1.0"
minGameVersion: "100.3"
dependencies: [ ]

说明:

  • name 会用来引用你的mod，慎重地去命名来避免和其他mod的名字重复
  • description 是mod加载界面会显示的文本，尽量的写简洁扼要一点。
  • dependencies 是非必须的;如果你想知道更多，去看前置 部分.
  • [new #2c61fcdfa] minGameVersion 是主游戏最小支持版本.

1.4 内容

[新特性 2d4270406] 现支持 .hjson 后缀

在你的mod的根目录你会有一个 content/ 目录，这里是所有json文件存储的地方，各种type会有不一样的子文件夹，在这个文件夹里面有如下子文件夹:

  • content/items/ 用来放 物品, 比如 copper 和 surge-alloy;
  • content/blocks/ 用来放 方块, 比如炮塔和地板;
  • content/mechs/ 用来放 玩家机甲, 比如 tau 和 glaive;
  • content/liquids/ 用来放 液体, 比如 water 和 slag;
  • content/units/ 用来放飞行或地面 单位类, 比如 reaper 和 dagger;
  • content/zones/ 用来放 区域, 战役地图的配置文件.

这个非常重要，这种结构是游戏本体加载mod所读取和结构。里面的各个文件名也很重要，比如在 content/blocks/<name>.json 目录的 name 将被当作数据节读取

[ new #62b2b25ee474fcf44aa86832c1373b38e16d703d ] 在 content/<content-type>/* 目录下的文件可以放在子目录里面, 来帮助你分类, 例如:

  • content/items/metals/iron.hjson, 这个创建了 iron 的物品.

内容文件夹的json文件开头应该像这样: 

{
    "type": "物品的种类",
    "name": "物品的显示名",
    "description": "物品介绍."
}
接口 类型 描述
type String 内容物的种类.
name String 内容物的显示名称.
description String 内容物的显示介绍.

其他接口会在~种类~中提及。

1.5 种类

种类有很多接口,其中最重要的是 种类;这是一个内容分析器分析的关键节点,用来决定这个内容物的种类. 一个 Router 不能是 Turret 类, 因为它们完全不一样.

种类会互相 继承 ,比如 MissileBulletType 继承了 BasicBulletType, 你可以在 MissileBulletType 中使用 BasicBulletType 中的所有接口,比如 damage, lifetime 和 speed. 接口都是大小写敏感的，比如: hitSize 不等于 hitsize.

每个内容物能做什么取决于它们的种类,有一些种类几乎不能做任何东西,它们大多数都是一些会被继承的父类. 比如 Block.

种类 也可以指各个种类的接口的数据种类.种类可以指其他东西,比如 float 是一个你可以使用 0.3 为数据的种类.

在这里你可以看到, 最高级的种类是 Revenant, 但是 bullet is的种类是 BulletType 所以你可以使用 MissileBulletType, 因为 MissileBulletType 继承了 BulletType. 

{
    "type": "Revenant",
    "weapon": {
	"bullet": {
	    "type": "MissileBulletType",
	    "damage": 9000
	}
    }
}

1.6 科技树

像 type 一样的类，是 research 类，这个类也是根类. 

{
    "research": "duo"
}

这个方块的研究会放在 duo 这个方块后面, 如果你想把某方块研究放在自己mod里面的一个方块后面，就写 <block-name>, 跨mod调用需要在调用的方块前加那一个mod名字做前缀.

研究消耗公式是 30 + requirements * 6, requirements 是建筑耗材. (换句话来说你不能在 requirements 中更改 research cost )

1.7 贴图

制作贴图所需的软件, 是一个支持透明编辑的软件 (比如: PS). 方块的贴图大小是 32 * 大小, 所以一个 2x2 的方块将会需要 64x64 的贴图.贴图必须是 .png 格式，要32位深度.

贴图可以直接放在 sprites/ 文件夹.内容分析器会正确地分析它们,所以你可以凭感觉来组织它们.

内容物会根据自己的名字寻找贴图.比如 content/blocks/my-hail.json 的名字是 my-hail 然后 sprites/my-hail.png 也有 ~my-hail~这个名字 , 所以这个贴图会被内容物使用.

内容物可能会寻找多个贴图. my-hail 是一个炮塔, 所以它会寻找一个叫 <name>-heat 的贴图，换句话来说，它会寻找叫 my-hail-heat 的贴图.

你可以在这里找到所有原版贴图:

  • https://github.com/Anuken/Mindustry/tree/master/core/assets-raw/sprites

你需要知道的另外一个事情是，材质都会通过一个叫 anti-aliasing filters 的东西，这个东西会让贴图在游戏中显示的有少许不一样. – 这个过滤器会给炮塔贴图加一个特殊的黑色边,所有你在制作贴图的时候必须注意,留一圈透明的边框,比如: Ripple

[new #a5fbc0756] 要覆盖原版材质, 你可以把材质放 sprites-override/.

1.9 前置

你可以添加依赖mod在 mod.json 里面: 

{
    "dependencies": [
	"other-mod-name",
	"not-a-mod"
    ]
}

依赖mod的名字全小写，其中的空格用 - 代替, 比如 Other MOD NamE 变成 other-mod-name.

要调用其他mod的资源,你必须给资源加上其他mod的前缀:

  • other-mod-name-not-copper 将会引用 other-mod-name 中的 not-copper
  • other-mod-name-angry-dagger 将会引用 other-mod-name 中的 angry-dagger
  • not-a-mod-angry-dagger 将会引用 not-a-mod 中的 angry-dagger

1.12 脚本

[new #268f3cc3c]

Mindustry的脚本依赖 Rhino JavaScript. 你可以把脚本放在 scripts/ 文件夹里面加入你的mod. 使用内建的 extendContent 方法, 你可以用JS继承Java类, 用 allowed classes 来加入你的命名空间.

比如:

  • scripts/silo.js 

    // 创建一个冲击波特效
const siloLaunchEffect = newEffect(20, e => {

    // 颜色从白到灰渐变
    Draw.color(Color.white, Color.lightGray, e.fin());

    // 厚度从 3 到 0
    Lines.stroke(e.fout() * 3);

    // 渲染一个圆，半径 0 到 100
    Lines.circle(e.x, e.y, e.fin() * 100); 
});

// 创建方块
const silo = extendContent(Block, "scatter-silo", {

    // 覆盖建筑设置
    buildConfiguration(tile, table) {
	table.addImageButton(
	    Icon.arrowUpSmall, 
	    Styles.clearTransi,

	    // 设置此方块的信号
	    // 点击 (此步骤同步客户端至服务端)
	    run(() => tile.configure(0))
	).size(50);
    },

    // 重写控制事件
    configured(tile, value) {

	// 确保弹药充足
	if (tile.entity.cons.valid()) {

	    // 使弹药在此方块处发射
	    Effects.effect(siloLaunchEffect, tile);

	    // 创建10个子弹
	    // 随机角度，速度和存在时间
	    for (var i = 0; i < 10; i++) {
		Calls.createBullet(
		    Bullets.flakExplosive, 
		    tile.getTeam(), 
		    tile.drawx(), 
		    tile.drawy(), 
		    Mathf.random(360), 
		    Mathf.random(0.5, 1.0), 
		    Mathf.random(0.2, 1.0)
		);
	    }

	    // 消耗物品
	    tile.entity.cons.trigger();
	}
    }
});

  • content/blocks/scatter-silo.hjson

description: "A player-activatable block that scatters bullets everywhere upon use."

category: turret
size: 2

update: true

		"titanium/30" ]

1.13 GitHub

你做了一些mod, 并且想发布它, 或者箱多人协力完成mod, 你可以选择使用 GitHub. 如果你完全不知道什么是 Git (或 GitHub) , 你可以用 GitHub Desktop, 或者使用其他你喜欢的工具.

你只需要知道怎么在Github创建仓库, 进度的变更都在本地目录, 你需要将其推送到Github上. 如果你的项目在Github上, 这里有三个方法分享它:

  • 用仓库名, 比如 Anuken/ExampleMod, 这样可以在游戏内拉取, 与下载;
  • 用压缩包, 比如 https://github.com/Anuken/ExampleMod/archive/master.zip, 这里下载一个压缩包, 然后放在游戏mod目录 (无需解压);
  • 加上 mindustry-mod 这个tag在你的仓库, 这可以让 #mods频道 的Discord机器人识别并添加至mod列表.

1.14 FAQ

  • time 在游戏中用 ticks 计算;
  • ticks 有时叫 帧数, 是 60/1 秒;
  • range 和其他距离都是 1/10 格;
  • 计算子弹范围需要用 lifetime 和 speed ，公式是 lifetime * speed = range;
  • 什么是 abstract?关于此类你要知道的,只有这是Java一个修饰符,代表这个类不能直接被加载.如果你强行加载会出现 "initialization exception" 诸如此类的错误;
  • 什么是 NullPointerException?这是一个有一些类中的必须接口没有设置的信息;
  • 什么是 bleeding-edge? 这个是开发版的用Github上面的master分支构建.其中的新特性会在下一个正式版本中使用.

2.16 属性

属于一个抽象物品的一个组 属性. 在 地板 中使用来给地板一个特殊属性, 比如 热量 和 孢子度 来给某些机器提效, 比如泵和培养机.

属性组 有4个属性:

  • 第 0 个是 热量,
  • 第 1 个是 孢子度,
  • 第 2 个是 含水度,

  • 第 3 个是 含油度.

    举个例子, 当你给一个地板 100 热量, 1 孢子度, 0.5 含水度和 0.1 含油度. 

    {
    "array": [ 100, 1, 0.5, 0.1]
}

    你可以把这个放 地板 类的文件里面，比如: 

    {
    "type": "Floor",
    "name": "magma",
    "attributes": { "array": [ 0.75, 0, 0, 0 ] }
}
